Project-1

Airbnb SQL Data Analysis

This project was developed as part of the course "Foundations of SQL: Learning the Data Language" by Data Tutor. It focuses on analyzing Airbnb data using SQL, with the aim of extracting valuable business insights.

The Problem/Opportunity:

There is a need to understand trends in pricing, availability, and listing quality on Airbnb in order to support strategic decision-making by hosts, tourism businesses, and local stakeholders.

Step 1: Creating the Database Schema

A relational schema was designed and created (listings, hosts, calendar, reviews). Constraints and primary keys were applied using CREATE TABLE.

Step 2: Data Import

Data from Inside Airbnb was loaded into SQL SSMS using COPY statements.

Step 3: SQL Query Development

Queries were written to extract KPIs such as average price per neighborhood, available days per listing, and review analysis.

Key Insights:

Pricing Trends

  • Price per Neighborhood: Significantly higher prices were observed in central city areas (e.g., Athens Center) compared to the suburbs.

  • Type of Accommodation: Entire homes are priced nearly twice as high as private rooms.

Availability Patterns

  • Availability: Listings with 300+ available days per year are typically operated by professional businesses.

Review Analysis

  • Average Rating: 4.6/5. Superhosts consistently receive better reviews.

The Outcome:

The project delivers actionable insights for hosts and startups in the short-term rental market. All questions were addressed through SQL in PostgreSQL, and findings were also visualized as part of the technical presentation.
